# 待办清单

> 记录项目开发中待办事项清单。
> 目的：方便任务安排，防止任务堆积和遗忘。
> 更新规则：确认已经完成的直接删除，不做留存处理。发现重复的待办项只保留一个，禁止出现重复
> 维护者：Claude Code
---

## 待办清单格式

```markdown
## [待办事项]
**背景：** [事项的背景]
**描述：** [简单介绍需要做的事情]
**类型：** [待办事项的类型：新功能｜修复BUG｜优化 ]
```

## 附件删除强一致性处理

**背景：** fix-02 代码评审（2026-04-14）发现，当前 `removeAttachment` 遇非 ENOENT 文件错误时仅 warn 后继续更新数据库，仍可能产生孤儿文件
**描述：** 可考虑：① 非 ENOENT 错误时中断操作抛出 500；或 ② 记录到死信队列/表，由定时任务重试删除，实现最终一致性。评审者已确认当前"尽力删除并放行"策略在现阶段可接受。
**类型：** 优化
